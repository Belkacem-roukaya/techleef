EdgeForge Industrial Gateway

TP complet Yocto pour BeagleBone

Type de projet : Linux embarque industriel

Carte cible : BeagleBone

Technologies : Yocto, KAS, BitBake, systemd, MQTT/REST, RAUC

Objectif : construire une distribution Linux industrielle avec application de gateway, images dev/prod, monitoring et OTA.

1. Vision du projet

Ce TP reprend la demarche du cahier des charges Yocto : structure projet, build, machine, distribution, image, users/groups, SDK, OTA et securite. On ajoute une vraie orientation industrielle : une gateway embarquee capable de lire des donnees machine, les journaliser, les publier via MQTT ou REST, surveiller son etat et se mettre a jour avec RAUC.

Nom du projet : EdgeForge Industrial Gateway

Application principale : edgeforge-agent

Distribution : edgeforge-distro

Images : edgeforge-image-dev et edgeforge-image-prod

Objectif

Fichier

SSTATE_DIR = "/work/sstate-cache"

TP 9 - Service systemd industriel

Objectif

Rendre l'application robuste et demarree automatiquement.

Exemple de service

[Unit]
Description=EdgeForge Industrial Agent
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=edgeuser
ExecStart=/usr/bin/edgeforge-agent --config /etc/edgeforge/agent.conf
Restart=always
RestartSec=5
WatchdogSec=30

[Install]
WantedBy=multi-user.target

Tests

systemctl status edgeforge-agent
journalctl -u edgeforge-agent -f
