EDGEFORGE INDUSTRIAL GATEWAY
TP complet Yocto pour BeagleBone

Type de projet : Linux embarque industriel
Carte cible : BeagleBone
Technologies : Yocto, KAS, BitBake, systemd, MQTT/REST, RAUC

Objectif :
Construire une distribution Linux industrielle avec Yocto pour BeagleBone,
integrant une application de gateway, des images dev/prod, du monitoring,
une partition data, un SDK, des mises a jour OTA et une securite progressive.


1. VISION DU PROJET

Ce TP reprend la demarche du cahier des charges Yocto :

1. Project structure
2. Build
3. Machine
4. Distribution
5. Image
6. Users and Groups
7. SDK
8. OTA
9. Security

On ajoute une orientation industrielle :

- application metier : edgeforge-agent
- service systemd robuste
- logs journald
- watchdog
- monitoring
- diagnostic CLI : edgeforgectl
- communication MQTT ou REST
- partition data
- separation image dev / image prod
- mise a jour OTA avec RAUC

  Images, application industrielle, users, packagegroups et services.

5. TP 1 - BUILD AVEC KAS

Fichier :

Layers a integrer :

- bitbake
- meta-openembedded
- meta-ti ou layer compatible BeagleBone

SSTATE_DIR = "/work/sstate-cache"

INHERIT += "create-spdx"

13. TP 9 - SERVICE SYSTEMD INDUSTRIEL

Objectif :
Rendre l'application robuste et demarree automatiquement.

Exemple de service :

[Unit]
Description=EdgeForge Industrial Agent
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=edgeuser
ExecStart=/usr/bin/edgeforge-agent --config /etc/edgeforge/agent.conf
Restart=always
RestartSec=5
WatchdogSec=30

[Install]
WantedBy=multi-user.target

Tests :

systemctl status edgeforge-agent
journalctl -u edgeforge-agent -f

24. CONSEIL IMPORTANT

Ne pas tout faire en meme temps.

Construire d'abord une image qui boot. Ensuite ajouter une seule capacite a la fois.
Chaque TP doit se terminer par un test clair sur la BeagleBone ou dans l'environnement
de build.

La bonne logique est :

1. build
2. boot
3. test
4. ajout d'une fonctionnalite
5. rebuild
6. validation
7. documentation
